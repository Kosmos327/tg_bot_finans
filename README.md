# tg_bot_finans

Внутренняя бизнес-система на базе Telegram Mini App с ролевым доступом, FastAPI-бэкендом и Google Sheets.

---

│   ├── main.py             # Точка входа приложения

│   ├── dependencies.py     # FastAPI зависимости (авторизация)

## Быстрый старт

### 1. Клонировать и установить зависимости

```bash
pip install -r requirements.txt
```

### 2. Настроить переменные окружения

Скопировать `.env.example` в `.env` и заполнить значения:

```bash
cp .env.example .env
```

```env
BOT_TOKEN=<токен Telegram-бота>
SPREADSHEET_ID=<ID Google Таблицы>
GOOGLE_CREDENTIALS_FILE=credentials.json
MINI_APP_URL=https://your-domain.com/app
```

### 3. Настроить Google Sheets

Создайте Google Таблицу с тремя листами:

#### Лист «Учёт сделок»
Столбцы (A–S):

| Столбец | Поле |
|---------|------|
| A | ID сделки |
| B | Статус сделки |
| C | Направление бизнеса |
| D | Клиент |
| E | Менеджер |
| F | Начислено с НДС |
| G | Наличие НДС |
| H | Оплачено |
| I | Дата начала проекта |
| J | Дата окончания проекта |
| K | Дата выставления акта |
| L | Переменный расход 1 |
| M | Переменный расход 2 |
| N | Бонус менеджера % |
| O | Бонус менеджера выплачено |
| P | Общепроизводственный расход |
| Q | Источник |
| R | Документ/ссылка |
| S | Комментарий |

#### Лист «Настройки» — роли пользователей

| A: telegram_user_id | B: full_name | C: role | D: active |
|---------------------|--------------|---------|-----------|
| 123456789 | Иван Иванов | manager | 1 |
| 987654321 | Мария Петрова | accountant | 1 |
| 111222333 | Алексей Сидоров | operations_director | 1 |
| 444555666 | Елена Кузнецова | head_of_sales | 1 |

Значения столбца `active`: `1`, `true`, `yes` или `да` — активный пользователь.

#### Лист «Журнал действий»

## Роли и права доступа

### Настройка ролей

Роль пользователя определяется по его `telegram_user_id` в листе **«Настройки»**.

Если пользователь не найден или `active ≠ 1`, ему показывается экран «Нет доступа».

### Доступные роли

| Роль | Код | Описание |
|------|-----|----------|
| Менеджер | `manager` | Создаёт и ведёт собственные сделки |
| Бухгалтер | `accountant` | Управляет оплатами и расходами |
| Операционный директор | `operations_director` | Полный доступ и аналитика |
| Руководитель отдела продаж | `head_of_sales` | Контроль команды, воронка, KPI |

## API эндпоинты

Все запросы требуют заголовок `X-Init-Data` с Telegram Web App initData.

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/me` | Текущий пользователь + роль + права |
| GET | `/dashboard` | Дашборд с учётом роли |
| GET | `/dashboard/manager` | Дашборд менеджера |
| GET | `/dashboard/accountant` | Дашборд бухгалтера |
| GET | `/dashboard/operations` | Дашборд операционного директора |
| GET | `/dashboard/sales` | Дашборд РОП |
| GET | `/deals/user` | Сделки текущего пользователя |
| GET | `/deals/all` | Все сделки (не для менеджера) |
| GET | `/deals/{deal_id}` | Конкретная сделка |
| POST | `/deals/create` | Создать сделку |
| PUT | `/deals/{deal_id}` | Обновить сделку |
| GET | `/journal/recent` | Последние записи журнала |
| GET | `/settings` | Список пользователей (для директора и РОП) |

## Дизайн Mini App

Каждая роль получает:
- Персональный заголовок с именем и значком роли
- KPI-карточки
- Кнопки быстрых действий
- Сегментированные вкладки
- Ролевые цвета оформления:
  - **Менеджер** — синий (`#2481cc`)
  - **Бухгалтер** — зелёный (`#2e7d32`)
  - **Операционный директор** — тёмно-серый (`#37474f`)
  - **РОП** — фиолетовый (`#6a1b9a`)

## Безопасность

- Все запросы к API проверяют Telegram HMAC-подпись initData
- Backend принудительно ограничивает редактирование полей по роли
- Менеджер видит только собственные сделки
- Frontend дублирует ограничения (поля скрыты/заблокированы), но backend является последней линией защиты
